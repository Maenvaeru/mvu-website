# MVU AI LAB - Официальный сайт визитка

Это проект на базе [Next.js](https://nextjs.org), созданный с использованием `create-next-app`.

## 🚀 Как запустить проект

1. Убедитесь, что у вас установлены Node.js (желательно версии 18+) и npm.
2. Для удобного запуска просто дважды кликните по файлу **`run.bat`** в корне проекта.
3. Если вы предпочитаете использовать терминал:
   ```bash
   npm run dev
   ```
4. Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## 🖼 Замена изображений и кэширование

**Важно:** Если вы просто заменяете файл картинки в папке `public/images/` на другой с тем же именем, изменения могут не отобразиться на сайте сразу. Это происходит из-за **интеллектуального кэширования** в Next.js (компонент `<Image>` оптимизирует картинки и кэширует их результат производительности ради).

### Как правильно обновлять изображения:
- **Способ 1 (Рекомендуемый):** Загрузите картинку с **новым именем** (например, `photo_v2.webp`) и просто измените путь к ней в коде (`src/app/page.tsx`). Это гарантирует, что браузеры ваших пользователей не будут показывать старую закэшированную картинку.
- **Способ 2 (Очистка кэша сборки):** Если вы оставили тоже самое имя файла и хотите его обновить, остановите локальный сервер, удалите скрытую папку `.next` в корне проекта (или хотя бы `.next/cache`), а затем снова запустите `run.bat`.
- **Способ 3 (Без сборки):** Иногда помогает просто открыть картинку напрямую в браузере (например `localhost:3000/images/my_photo.jpg`) и нажать `Ctrl+F5` для жесткого сброса кэша браузера.

## 🛠 Разработка

Основная редактируемая страница сайта находится здесь: `src/app/page.tsx`.
Любые сохранения этого файла будут автоматически перезагружать страницу в браузере без потери состояния (Hot Fast Reload).

Для стилизации используется современный **Tailwind CSS v4** и компоненты **Framer Motion** для плавных анимаций переходов.

## 🏗 Архитектура и Развитие проекта (Результат исследования)

Этот раздел содержит рекомендации по дальнейшему развитию, хостингу, выбору CMS и платежной системы (согласно проведеному Multi-Agent Brainstorming). 
**👉 Полная и профессиональная инструкция по развертыванию сайта статики на Cloudflare Pages (без GitHub), а также подробнейшая интеграция CMS Outstatic и платежной системы Robokassa находится в файле [DEPLOYMENT.md](./DEPLOYMENT.md).**

### 1. Выбор бесплатного хостинга для Next.js
*   **Vercel (Hobby Tier):** Самый удобный и нативный вариант для Next.js. Поддерживает SSR, кэширование и Edge Functions. *Внимание:* правилами запрещено коммерческое использование (продажа товаров), но для MVP или сайта-визитки подойдет отлично.
*   **Cloudflare Pages:** Отличная альтернатива с глобальным быстрым кэшированием (очень актуально для РФ). Не имеет жестких ограничений на коммерческие сайты в бесплатном тарифе. Допускает интеграцию с Cloudflare Workers для бэкенда.
*   *Рекомендация:* Использовать **Cloudflare Pages**, поскольку проект будет принимать платежи. При масштабировании или добавлении тяжелой БД (Postgres) имеет смысл перейти на дешевый VPS (Timeweb/Selectel ~250 руб/мес) + Coolify (бесплатный аналог Vercel).

### 2. Админ-панель (CMS) для контент-менеджмента
Для управления блоками (например, добавление проектов в портфолио или постов на главную страницу) рекомендуются следующие варианты (все имеют отличные бесплатные планы):
*   **Outstatic:** Идеальный вариант. Эта CMS встраивается прямо в структуру Next.js (`/outstatic`). Данные хранятся в вашем GitHub репозитории в виде Markdown/JSON. Не требует сторонней БД. Зависимость "нуль".
*   **Sanity.io:** Очень мощная Headless CMS. Данные структурированы, удобное хранение медиа, отличные бесплатные лимиты. Потребует создания отдельной папки `/studio` для интерфейса автора.
*   **Notion API:** Можно использовать страницы Notion как базу данных для сайта (например, через `react-notion-x`). Огромный плюс — нулевой порог входа для нетехнических редакторов.
*   *Рекомендация:* **Outstatic** для максимальной скорости интеграции, отсутствия платежей за БД и хранения "всего в одном месте".

### 3. Интеграция платежной системы
*   **Для РФ (Самозанятые / ИП / ООО):**
    *   *ЮKassa (YooKassa):* Отлично документированное API, простая интеграция библиотек (даже неофициальных) в Next.js API Routes (`src/app/api`).
    *   *Robokassa / Prodamus:* Альтернативные платформы, которые иногда легче подключаются для инфобизнеса.
*   **Для микротранзакций / Псевдо-оплаты:**
    *   Если полноценный эквайринг пока не нужен, можно принимать донаты или платежи за услуги (подписки) через уже существующий в проекте **Boosty** (кнопка перенаправления).
    *   *Telegram Payments:* Быстрый запуск чекаута через бота.
