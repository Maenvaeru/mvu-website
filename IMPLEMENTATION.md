# Журнал реализации исправлений MVU Website

## Ошибка: Задержка обновления сайта на GitHub Pages (06.03.2026)
**Симптомы:**
- Пользователь внес изменения в тексты через локальную админку, нажал деплой, но сайт не обновился.
- На сайте виден баг `[object Object]` в заголовке Hero.
- Мобильное меню отображает крестик, но контент меню не виден.

**Анализ:**
1. `git log` показал, что ветка `gh-pages` не обновлялась 3 дня (с 3 марта).
2. `git status` показал, что все последние фиксы (включая исправление `[object Object]` и мобильное меню) находятся в статусе `Modified` / `Untracked` и не закоммичены.
3. Деплой через `gh-pages -d out` выполняется локально. Если локальная сборка `out` была старой или содержала ошибки кэша, деплой просто запушил старый код.
4. Баг `[object Object]` возникает, когда React пытается отрендерить JS-объект как текст. Вероятно, данные из `homepage.md` (поле `heroTitle`) или динамические блоки приходят как объекты.

**Решения:**
1. Сделать функцию `renderHeroTitle` максимально защищенной от не-строковых данных.
2. Проверить `layout.tsx` на предмет `overflow-hidden` в хедере, который мешает абсолютному позиционированию меню.
3. Закоммитить все изменения в `master`.
4. Выполнить чистую сборку `npm run build` и деплой.
