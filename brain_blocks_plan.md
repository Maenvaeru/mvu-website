# План расширения функционала админки

Для реализации возможности «добавлять блоки» нам нужно изменить способ хранения данных в `homepage.md` и способ их отображения.

## Предлагаемая архитектура: «Гибкие секции»

### 1. Изменение формата данных
Вместо фиксированных полей `service1...`, `service2...`, мы введем массив `sections` в JSON-поле внутри Markdown.
Однако Outstatic плохо работает с массивами объектов в интерфейсе. 

**Решение:** 
Создадим в `HomepageEditor` динамическую форму, которая сохраняет список блоков в одно текстовое поле `blocksData` в формате JSON.

### 2. Поддержка типов блоков
- **Услуги (Services)**
- **Преимущества (Features)**
- **Наши работы (Portfolio)**
- **FAQ**
- **Произвольный HTML/Markdown**

### 3. Компоненты
- `BlockManager`: контейнер в админке для добавления/удаления/сортировки блоков.
- `DynamicRenderer`: компонент на фронтенде, который проходит по массиву `blocksData` и рендерит нужные секции.
