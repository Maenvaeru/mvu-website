# Инструкция по управлению сайтом MVU AI LAB

Этот документ содержит инструкции по редактированию контента и работе с платежной системой в условиях статического хостинга на **GitHub Pages**.

---

## 1. Редактирование контента (Админка)

Так как GitHub Pages — это статический хостинг, на самом сайте (по адресу `.github.io`) «живой» админки быть не может. Все изменения делаются локально, а затем публикуются.

### Как редактировать тексты и блоки:
1.  **Локальный запуск**:
    Откройте терминал в папке проекта и запустите:
    ```bash
    npm run dev
    ```
    Сайт будет доступен по адресу `http://localhost:3000`.

2.  **Прямое редактирование**:
    - Основной текст главной страницы находится в файле: `src/app/page.tsx`.
    - Общий макет (шапка и футер): `src/app/layout.tsx`.
    - Все изображения лежат в папке: `public/images/`.

3.  **Использование Outstatic (в процессе интеграции)**:
    Outstatic позволяет редактировать контент через удобный интерфейс. 
    - Для этого нужно зайти в админку локально (после завершения настройки).
    - Все изменения сохраняются в виде файлов в проекте.

### Как опубликовать изменения:
После любого редактирования кода или контента, выполните в терминале одну команду:
```bash
npm run deploy
```
Она автоматически:
1. Соберет проект (`next build`).
2. Добавит необходимые настройки для GitHub (`.nojekyll`).
3. Отправит обновленную версию на сайт. Ваш сайт обновится в течение 1-2 минут.

## 2. Оплата и Robokassa

GitHub Pages не поддерживает серверный код (Node.js API), поэтому Robokassa не может отправить «уведомление об успешной оплате» (ResultURL) напрямую на ваш сайт.

### Что изменилось:
1.  **Логика оплаты**: Мы будем использовать **Cloudflare Workers**. 
    - Ваш сайт на GitHub Pages будет отправлять пользователя на оплату.
    - Robokassa будет отправлять подтверждение на отдельный микросервис в Cloudflare (который бесплатен и не блокируется).
2.  **Настройки в кабинете Robokassa**:
    - **ResultURL**: Должен вести на адрес вашего Cloudflare Worker (будет настроен далее).
    - **SuccessURL / FailURL**: Могут вести обратно на ваш сайт GitHub Pages (например, `https://maenvaeru.github.io/mvu-website/success`).

### Что нужно сделать для запуска платежей:
1.  Завершить регистрацию магазина в Robokassa.
2.  Передать мне (или прописать в `.env`) `MerchantLogin`, `PASS1` и `PASS2`.
3.  Я разверну маленькую «приемную» для платежей в Cloudflare.

## 3. Полезные команды

- `npm run dev` — запуск сайта на вашем компьютере для правок.
- `npm run deploy` — публикация всех правок в интернет.
- `git add .` + `git commit -m "описание"` — сохранение истории изменений.

> **Статус проекта**: Сайт готов, доступен в РФ. На следующем этапе мы можем завершить интеграцию Outstatic и Robokassa.
